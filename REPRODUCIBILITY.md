# Reproducibility

This document specifies the exact environment, models, seeds, and commands
needed to regenerate every numeric result in the accompanying paper. If a
table cell in the paper is currently marked `\todo{}`, it is because the
corresponding run has not yet been executed; running the commands below
on the specified hardware will produce a `trials.jsonl` whose contents
flow into that cell automatically via `analysis/build_tables.py`.

---

## 1. Hardware

The reference machine for the headline numbers is:

- 13th Gen Intel Core i5-13420H (8C/12T, 2.10 GHz base)
- 16 GB DDR5
- NVIDIA GeForce RTX 4050 Laptop GPU, 6 GB VRAM
- Windows 11, 64-bit

The framework runs on any Linux/macOS/Windows host with Python 3.11+. A
GPU is recommended for local Ollama; without one, use the HF fallback
config (slower; rate-limited).

## 2. Software

- Python 3.11.x
- Ollama (host install — see https://ollama.com/download)
- LaTeX (TeX Live or MiKTeX) for the paper

```bash
# Python deps
pip install -e .

# Models (pull once; sizes are q4_K_M weights on disk)
# --- default config ---
ollama pull qwen3:4b-instruct-2507-q4_K_M           # ~2.5 GB; planner/reviewer/explainer
ollama pull qwen2.5-coder:7b-instruct-q4_K_M        # ~4.5 GB; generator/optimizer

# --- strong (opt-in; MoE; spills to RAM) ---
ollama pull qwen3-coder:30b-a3b-instruct-q4_K_M     # ~17 GB on disk; 3B active params

# --- cross-model (independent vendor family) ---
ollama pull llama3.1:8b-instruct-q4_K_M             # ~4.9 GB
ollama pull deepseek-coder-v2:16b-lite-instruct-q4_K_M  # ~9 GB; 2.4B active

# Confirm the daemon is reachable
curl http://localhost:11434/api/tags
```

All models are open-weight (Apache-2.0 / Llama-3 community licence). **No closed-weight or paid-API model is used in any reported result.**

Pinned model digests at the time of submission are recorded in
`results/<run_id>/run_meta.json`.

## 3. Seeds

Each (problem, condition) is repeated for `N` trials with seeds
`{0, 1, ..., N-1}`. Seeds flow through to:
1. The Ollama generator's `seed` option (where the model server honours it).
2. The LLM-generated test sampler.
3. The trial runner's iteration order (deterministic given the seed).

GPU floating-point non-determinism in the Ollama runtime means bit-exact
reproducibility is not guaranteed across hardware; observed run-to-run
variation in pass-rate at fixed seed is bounded by the reported standard
deviations and does not change the qualitative conclusions.

## 4. Run

The full experimental grid (sections IV.A–IV.G of the paper) is regenerated by:

```bash
# Run the full grid (~30-45 hours on the reference RTX 4050).
RUN_ID=submission-2026 ./scripts/run_full_grid.sh

# Build the paper tables.
python -m execugraph.analysis.build_tables \
    results/submission-2026 --out ../paper/tables
```

Individual experiments can be run on their own:

```bash
# E1: internal-30 baselines + multi-full, 5 trials each.
python -m execugraph.runner.cli --config configs/default.yaml \
    --output results/e1_so --condition single-oneshot --n-trials 5
python -m execugraph.runner.cli --config configs/default.yaml \
    --output results/e1_sr --condition single-retry  --n-trials 5
python -m execugraph.runner.cli --config configs/default.yaml \
    --output results/e1_mf --condition multi-full    --n-trials 5
```

Replace `--config configs/default.yaml` with `configs/hf_fallback.yaml` to
use the HuggingFace Inference API backend, with `configs/strong.yaml` for
the slower-but-stronger Qwen3-Coder-30B-A3B MoE generator, or with
`configs/crossmodel.yaml` for the cross-model condition (Llama-3.1-8B +
DeepSeek-Coder-V2-Lite-16B).

For a 5-minute smoke test (no real LLM, just the harness):

```bash
python tests/integration/run_smoke_no_llm.py
```

## 5. Per-trial artifacts

Every trial appends one JSON line to `results/<run_id>/trials.jsonl`
with the schema specified in `execugraph/runner/trial_runner.py`. A small
synthetic example is committed under `results/example_run/` so the
table-generation pipeline can be exercised without an LLM. **The numbers
in `results/example_run/` are synthetic and must not be cited; see
`results/example_run/README.md`.**

## 6. Verification

```bash
pytest -q tests/unit                 # 31 tests, no LLM needed
ruff check execugraph tests          # static checks
python -m execugraph.analysis.build_tables results/example_run \
    --out /tmp/tables_demo            # smoke the table generator
```
